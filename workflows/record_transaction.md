# Workflow: บันทึกรายการขาย

## ประเภทรายการและการตั้งค่า

| สถานการณ์ | ประเภท | สถานะบิล | สถานะจ่าย | หมายเหตุ |
|---|---|---|---|---|
| ขายและจ่ายทันที | ขายปกติ | เปิดบิลแล้ว | จ่ายแล้ว | ปิดรายการได้เลย |
| สั่งแล้วค้างจ่าย | ขายปกติ | เปิดบิลแล้ว | ค้างจ่าย | ยังไม่ได้รับของ รอจ่ายก่อน |
| เบิกของไปก่อน ยังไม่มีบิล | เบิกของก่อน | ยังไม่เปิดบิล | ค้างจ่าย | ระบบบันทึกว่ารับของแล้ว ค้างเงิน |

## ขั้นตอน Tab 1: บันทึกรายการ

1. เปิด Tab **📋 บันทึกรายการ**
2. เลือก วันที่ / ลูกค้า / สินค้า / จำนวน
3. ระบบแสดงราคารวมและคะแนนให้อัตโนมัติ
4. เลือก ประเภท / สถานะบิล / สถานะจ่าย ตามตารางด้านบน
5. กด **บันทึก**

## ขั้นตอน Tab 2: รับของ/จ่ายเงินบางส่วน

ใช้เมื่อลูกค้ามาจ่ายเพิ่มหรือมารับของ หลังจากบันทึกรายการไว้แล้ว

1. เปิด Tab **💰 รับของ / จ่ายเงิน**
2. เลือกลูกค้า → ระบบแสดงรายการค้างทั้งหมด
3. เลือกรายการที่ต้องการบันทึก
4. ดูตัวเลขสรุป:
   - **จ่ายแล้วสะสม** — รวมทุกครั้งที่จ่ายมา
   - **รับได้อีก** — ชิ้นที่รับได้ = floor(จ่ายสะสม ÷ ราคา/ชิ้น) − รับไปแล้ว
5. เลือกประเภท:
   - **รับของ** — ลูกค้ามารับของ (ไม่จ่ายเงินเพิ่ม)
   - **จ่ายเงิน** — ลูกค้าจ่ายเงิน (ไม่รับของ)
   - **ทั้งคู่** — จ่ายและรับพร้อมกัน
6. กรอกจำนวน → กด **บันทึก**
7. ถ้ารับของเกินยอดจ่าย ระบบจะ error — ต้องบันทึกจ่ายเงินก่อน

## กฎสำคัญ

> **รับของได้ = floor(ยอดจ่ายสะสมทั้งหมด ÷ ราคา/ชิ้น)**
>
> ตัวอย่าง: สินค้าราคา 500 บาท / จ่ายมาแล้วรวม 1,200 บาท → รับได้ 2 ชิ้น (ไม่ใช่ 2.4)

## การดูยอดค้าง

- Tab **📊 ยอดค้าง** แสดงรายการที่ยังค้างอยู่ทั้งหมด
- กรองต่อลูกค้าได้
- ด้านล่างแสดงสรุปรวมต่อลูกค้า (เมื่อดูทั้งหมด)
